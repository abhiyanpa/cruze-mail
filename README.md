# 🚀 CRUZE MAIL

**CRUZE MAIL** is a lightning-fast, highly secure, serverless disposable email service. It allows users to instantly generate temporary email addresses to protect their primary inboxes from spam, phishing, and unwanted newsletters.

Built with a state-of-the-art serverless architecture, it leverages Cloudflare Workers for instant email routing and Firebase Firestore for real-time inbox synchronization.

---

## ✨ Features

- **Instant Generation**: Create a new, randomized disposable email address in one click.
- **Real-Time Delivery**: Incoming emails appear instantly in the UI without needing to refresh the page.
- **Multi-Domain Support**: Generate emails across multiple custom domains.
- **Secure Architecture**: All session tokens are encrypted using AES-256-GCM. The frontend holds zero backend secrets.
- **Auto-Destruction**: Inboxes automatically expire and self-destruct after 24 hours to ensure absolute privacy.
- **Inbox Renewal**: Extend the lifespan of an active inbox up to a maximum of 48 hours.
- **User Authentication**: Optional Google Sign-In powered by Firebase Auth.
- **Permanent Saved Inboxes**: Authenticated users can save up to 5 permanent inboxes to their Dashboard that never expire.
- **Built-in Rate Limiting**: Anti-abuse mechanisms protect the backend from spam creation.

---

## 🛠️ Technology Stack

- **Frontend**: Vanilla JS, HTML5, CSS3. Hosted on [Vercel](https://vercel.com).
- **Backend / API**: [Cloudflare Workers](https://workers.cloudflare.com/) (Serverless compute).
- **Email Parsing**: `postal-mime` (Cloudflare Email Routing integration).
- **Database**: Google [Firebase Firestore](https://firebase.google.com/docs/firestore) (Real-time NoSQL).
- **Authentication**: Google [Firebase Authentication](https://firebase.google.com/docs/auth).

---

## 🏗️ Architecture & Security

CRUZE MAIL is designed with strict security boundaries:

1. **Cloudflare Email Routing**: Receives raw SMTP emails and forwards them instantly to the Cloudflare Worker.
2. **Cloudflare Worker**: 
   - Parses the raw MIME email.
   - securely verifies if the receiving address maps to a valid, active `inboxId`.
   - Writes the parsed email directly into the Firestore Database.
   - Enforces rate-limiting for new inbox generation.
3. **Frontend Application**:
   - Holds an **AES-256-GCM encrypted token** generated by the Worker. The frontend *cannot* decrypt this token.
   - Listens to Firestore in real-time (`onSnapshot`) using a securely scoped `inboxId` constraint.
   - Authenticates users via Google Auth to manage "Saved Inboxes".

## 🚀 Deployment Guide

### 1. Cloudflare Setup
1. Add your domains to Cloudflare and enable **Email Routing**.
2. Configure Catch-All routes to point to a Worker.

### 2. Firebase Setup
1. Create a Firebase project and enable **Firestore Database**.
2. Create a Service Account and download the JSON key.
3. Apply the following Firestore Security Rules:
```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /emails/{emailId} { allow read: if true; allow write: if false; }
    match /emailMap/{localPart} { allow read, write: if false; }
    match /users/{userId} { allow read, write: if request.auth != null && request.auth.uid == userId; }
  }
}
```
4. Enable **Google** in the Authentication Sign-In Providers.

### 3. Worker Deployment
1. Navigate to the `/worker` directory.
2. Authenticate with Wrangler: `npx wrangler login`
3. Deploy the worker and set secrets: `node deploy.cjs`

### 4. Frontend Deployment
1. Navigate to the `/frontend` directory.
2. Update `js/firebase-config.js` with your public Firebase configuration.
3. Deploy to Vercel: `npx vercel --prod`
